# Documentazione DEPLOIE

Questa cartella è la fonte ufficiale della memoria progettuale di DEPLOIE.

## Documenti principali

- `MASTER.md`: identità, principi e memoria cumulativa del progetto.
- `ROADMAP.md`: fasi, stato e prossime attività.
- `PROJECT_STATE.md`: stato sintetico da leggere all'inizio di ogni sessione.
- `DECISIONS.md`: registro delle decisioni approvate e congelate.
- `BACKLOG.md`: idee e attività rinviate, non abbandonate.
- `sessions/`: verbali cronologici delle sessioni.

## Regola operativa

Alla chiusura di ogni sessione si aggiornano almeno:

1. `PROJECT_STATE.md`;
2. `ROADMAP.md`;
3. un verbale dentro `sessions/`;
4. `DECISIONS.md` e `BACKLOG.md` quando necessario.

La documentazione viene aggiornata senza cancellare o riscrivere la storia del progetto. Il codice applicativo resta separato e non deve essere modificato durante i soli aggiornamenti documentali.
